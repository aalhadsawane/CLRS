## 34.3-1

> Verify that the circuit in Figure 34.8(b) is unsatisfiable.

For the output to be $1$, the final AND gate forces every one of its inputs to be $1$. In particular, the gate labeled $g_{13}$ must be $1$, which forces $x_1 = x_2 = 1$ and $x_3 = 0$. But then the other branch of the circuit requires the value of $x_3$ to be $1$. This contradiction shows that no assignment can satisfy the circuit.

## 34.3-2

> Show that the $\le_\text P$ relation is a transitive relation on languages. That is, show that if $L_1 \le_\text P L_2$ and $L_2 \le_\text P L_3$, then $L_1 \le_\text P L_3$.

Let $f$ be a polynomial-time reduction from $L_1$ to $L_2$, and let $g$ be a polynomial-time reduction from $L_2$ to $L_3$. Then $g \circ f$ is computable in polynomial time, since a composition of polynomial-time algorithms is still polynomial-time.

Also,
$$
x \in L_1 \iff f(x) \in L_2 \iff g(f(x)) \in L_3.
$$
Hence $g \circ f$ is a polynomial-time reduction from $L_1$ to $L_3$, so $L_1 \le_\text P L_3$.

## 34.3-3

> Prove that $L \le_\text P \bar L$ if and only if $\bar L \le_\text P L$.

Suppose $f$ reduces $L$ to $\bar L$. Then
$$
x \in L \iff f(x) \in \bar L.
$$
Taking complements,
$$
x \in \bar L \iff f(x) \in L.
$$
Thus the same function $f$ reduces $\bar L$ to $L$.

The converse is symmetric. Therefore $L \le_\text P \bar L$ iff $\bar L \le_\text P L$.

## 34.3-4

> Show that we could have used a satisfying assignment as a certificate in an alternative proof of Lemma 34.5. Which certificate makes for an easier proof?

Yes. Use as certificate an assignment to the input variables of the circuit. The verifier evaluates the circuit gate by gate in topological order and accepts iff the output gate evaluates to $1$. This takes polynomial time.

The certificate used in the text is easier for the proof: giving the values of all gates lets the verifier check each gate locally, instead of recomputing the whole circuit from the inputs.

## 34.3-5

> The proof of Lemma 34.6 assumes that the working storage for algorithm A occupies a contiguous region of polynomial size. Where in the proof do we exploit this assumption? Argue that this assumption does not involve any loss of generality.

We use this assumption when we encode each configuration of $A$ by listing the contents of its work cells in order, and when we build the circuit that computes the next configuration from the current one. The proof assumes that the bits of a configuration can be indexed as positions $1,2,\dots,p(n)$ in one block.

This causes no loss of generality. Any polynomial-space computation uses only polynomially many cells, so we can relabel the cells it ever uses as $1,2,\dots,p(n)$ and encode unused locations by blanks. Thus any verifier can be represented with a contiguous polynomial-size work area.

## 34.3-6

> A language $L$ is **_complete_** for a language class $C$ with respect to polynomial-time reductions if $L \in C$ and $L' \le_\text P L$ for all $L' \in C$. Show that $\emptyset$ and $\\{0, 1\\}^\*$ are the only languages in $\text P$ that are not complete for $\text P$ with respect to polynomial-time reductions.

Let $L \in \text P$ be neither $\emptyset$ nor $\{0,1\}^*$. Then there exist strings $y \in L$ and $z \notin L$.

Now let $L' \in \text P$. Since membership in $L'$ can be decided in polynomial time, define
$$
f(x)=
\begin{cases}
y, & x \in L',\\
z, & x \notin L'.
\end{cases}
$$
This function is computable in polynomial time, and
$$
x \in L' \iff f(x) \in L.
$$
So $L' \le_\text P L$. Hence every nontrivial language in $\text P$ is complete for $\text P$.

On the other hand, if $L=\emptyset$, then $L' \le_\text P L$ is possible only when $L'=\emptyset$. If $L=\{0,1\}^*$, then $L' \le_\text P L$ is possible only when $L'=\{0,1\}^*$. Thus these two languages are the only exceptions.

## 34.3-7

> Show that, with respect to polynomial-time reductions (see Exercise 34.3-6), $L$ is complete for $\text{NP}$ if and only if $\bar L$ is complete for $\text{co-NP}$.

Assume $L$ is complete for $\text{NP}$. Then $L \in \text{NP}$, so $\bar L \in \text{co-NP}$.

Let $K \in \text{co-NP}$. Then $\bar K \in \text{NP}$. Since $L$ is $\text{NP}$-complete, $\bar K \le_\text P L$. By Exercise 34.3-3, this implies $K \le_\text P \bar L$. Therefore every language in $\text{co-NP}$ reduces to $\bar L$, so $\bar L$ is complete for $\text{co-NP}$.

The converse is the same argument with complements reversed. Hence $L$ is $\text{NP}$-complete iff $\bar L$ is $\text{co-NP}$-complete.

## 34.3-8

> The reduction algorithm $F$ in the proof of Lemma 34.6 constructs the circuit $C = f(x)$ based on knowledge of $x$, $A$, and $k$. Professor Sartre observes that the string $x$ is input to $F$, but only the existence of $A$, $k$, and the constant factor implicit in the $O(n^k)$ running time is known to $F$ (since the language $L$ belongs to $\text{NP}$), not their actual values. Thus, the professor concludes that $F$ can't possibly construct the circuit $C$ and that the language $\text{CIRCUIT-SAT}$ is not necessarily $\text{NP-hard}$. Explain the flaw in the professor's reasoning.

The flaw is that, once we fix a language $L \in \text{NP}$, we also fix a particular polynomial-time verifier $A$ for $L$ and a particular polynomial bound $n^k$ on its running time. The reduction $F$ is allowed to depend on this fixed verifier and this fixed bound.

Thus $F$ does not have to discover $A$ and $k$ from the input $x$. They are part of the definition of the reduction for the chosen language $L$, and only $x$ varies from one input to the next.
